Every six months somebody redefines "agent" and the discourse resets. Let me give you a definition that has survived three production deployments.

> An agent is a loop. The loop has access to tools. The loop decides when to stop.

That's it. That's the whole thing. The rest is implementation detail.

## Four things that get called agents but aren't

1. A chatbot with RAG.
2. A workflow with an LLM node.
3. A single function call.
4. A wizard with branching.
